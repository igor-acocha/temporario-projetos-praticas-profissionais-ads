## TST008 Cadastro de Projeto na Plataforma

**Caso de uso que se baseia:** CDU008 Cadastrar Projeto  

**Cenário:** Cadastro de Projeto com Dados Válidos

**Requisitos associados:**
- RF045: O sistema deve permitir que o empreendedor cadastre novos projetos.  
- RF048: O sistema deve registrar o projeto no banco de dados.  
- RF049: O sistema deve exibir mensagem de sucesso confirmando o cadastro.  
- RF050: O sistema deve disponibilizar o projeto para visualização pelos estudantes.  
- RF051: O sistema deve permitir ao empreendedor cancelar o processo de cadastro antes de concluir.  
- RF052: Nenhuma informação deve ser salva em caso de cancelamento.  
- RF053: O sistema deve exibir mensagem de erro se houver campos obrigatórios não preenchidos.  

---

**Preparação**

a) O empreendedor deve estar logado na plataforma.  
b) O perfil do empreendedor deve estar completo e ativo.

**Passos para execução do teste**

1. O empreendedor acessa a plataforma TechStart.  
2. Navega até a opção “Cadastrar Projeto”.  
3. Preenche todos os campos obrigatórios: título e descrição.  
4. Clica em “Salvar”.  

---

**Resultado esperado**

a) O sistema valida os dados e exibe a mensagem:

> **“Projeto cadastrado com sucesso! Disponível para visualização pelos estudantes.”**

b) O projeto é registrado corretamente no banco de dados.  
c) O projeto torna-se visível para estudantes interessados na plataforma.  

---

**Cenário Alternativo: Cancelamento do Cadastro**

**Requisitos associados:**
- RF051: O sistema deve permitir ao empreendedor cancelar o processo de cadastro antes de concluir.  
- RF052: Nenhuma informação deve ser salva em caso de cancelamento.  

**Passos:**
1. Preencher parcialmente os campos do projeto.  
2. Clicar em “Cancelar” ou fechar a página antes de concluir.  

**Resultado esperado:**  
a) O sistema interrompe o processo de cadastro.  
b) Nenhuma informação é salva no banco de dados.  
c) O empreendedor é redirecionado à tela inicial ou ao painel de projetos.  

---

**Cenário de Exceção 1: Campos Obrigatórios Não Preenchidos**

**Requisitos associados:**
- RF053: O sistema deve validar todos os campos obrigatórios.  
- RF053: O sistema deve impedir o cadastro se houver campos obrigatórios vazios.  

**Passos:**
1. Clicar em “Salvar” sem preencher todos os campos obrigatórios.  

**Resultado esperado:**  
a) O sistema exibe mensagem de erro:  

> **“Preencha todos os campos obrigatórios antes de salvar.”**

b) O cadastro é interrompido até que os campos estejam devidamente preenchidos.
