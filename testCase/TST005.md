## TST005 Demonstrar Interesse em Projeto

**Caso de uso que se baseia:** CDU005 Demonstrar Interesse  

**Cenário:** Estudante demonstra interesse em um projeto disponível na plataforma

**Requisitos associados:**
- RF027: O sistema deve permitir que estudantes visualizem os projetos publicados por empreendedores.  
- RF028: O sistema deve permitir que estudantes demonstrem interesse em um projeto.  
- RF029: O sistema deve exigir o preenchimento do campo de descrição com no mínimo 300 caracteres.  
- RF030: O sistema deve registrar o interesse do estudante (inscrição) no projeto selecionado.  
- RF031: O sistema deve confirmar o envio e exibir mensagem de sucesso.

---

**Preparação**

a) Deve haver ao menos um projeto ativo publicado por um empreendedor.  
b) O estudante deve possuir cadastro ativo e estar autenticado na plataforma.

**Passos para Execução do Teste**

1. Acessar a plataforma TechStart com o login do estudante.  
2. Navegar até o menu “Projetos Disponíveis”.  
3. Visualizar a lista de projetos publicados.  
4. Selecionar o projeto “App de Gestão Financeira Júnior”.  
5. Clicar no botão “Tenho Interesse”.  
6. Inserir a descrição detalhada da proposta de solução (mínimo de 300 caracteres).  
7. Clicar em “Enviar”.

---

**Resultado Esperado**

a) O sistema valida o preenchimento e exibe a mensagem:

> **“Interesse registrado com sucesso! Sua ideia foi enviada para análise do empreendedor.”**

b) A inscrição é registrada no banco de dados, vinculando o estudante ao projeto.  
c) O status da inscrição é definido como “Enviado”.  
d) O empreendedor responsável pelo projeto passa a visualizar a nova inscrição em sua área de gerenciamento.

---

**Cenário Alternativo: Cancelamento do Envio**

**Requisitos associados:**
- RF032: O sistema deve permitir cancelar o processo de envio antes da conclusão.  
- RF033: Nenhuma informação deve ser salva caso o estudante cancele a ação.

**Passos para execução do teste:**

1. Estudante preenche parcialmente a descrição.  
2. Clica em “Cancelar”.

**Resultado esperado:**  
a) O sistema interrompe o processo e retorna à listagem de projetos.  
b) Nenhum dado é salvo no banco de dados.  
c) O status da inscrição permanece não enviado.

---

**Cenário de Exceção 1: Campo Obrigatório Não Preenchido**

**Requisitos associados:**
- RF034: O sistema deve manter o botão “Enviar” desabilitado até o campo ser preenchido corretamente.  
- RF035: O campo de descrição deve conter no mínimo 300 caracteres.

**Preparação:**  
O estudante acessa a tela e digita uma descrição com menos de 300 caracteres.

**Passos para execução do teste:**

1. Digitar uma ideia com 100 caracteres.  
2. Verificar o estado do botão “Enviar”.

**Resultado esperado:**  
a) O botão “Enviar” permanece desabilitado.  
b) O sistema exibe aviso:

> **“A descrição deve conter no mínimo 300 caracteres.”**

c) Nenhum dado é registrado no banco até que a validação seja satisfeita.
