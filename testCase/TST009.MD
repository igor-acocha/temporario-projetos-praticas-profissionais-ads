## TST009 Gerenciar projetos

**Caso de uso que se baseia:** CDU009 Gerenciar projetos

**Cenário:** Visualizar, Editar e Salvar Projeto

**Requisitos associados:**

* RF001: O sistema deve permitir que o empreendedor visualize seus projetos cadastrados.
* RF002: O sistema deve permitir que o empreendedor edite informações de seus projetos.
* RF003: O sistema deve permitir que o empreendedor exclua um projeto.
* RF004: O sistema deve salvar as alterações realizadas e atualizar a lista de projetos.
* RF005: sistema deve exibir mensagens de confirmação ou erro para cada ação realizada.

**Preparação:**

a) O empreendedor deve estar logado no sistema.
b) Deve existir pelo menos um projeto cadastrado para o usuário.

**Dados de teste:**

| Projeto               | Status         | Descrição                     | Data de Criação| Dono     |
| --------------------- | -------------- | ------------------------      | -----------    | -------  |
| App de Finanças       | Em andamento   | App para controle financeiro  | 01/09/2025     | Felipe   |
| Plataforma Educacional| Concluído      | Sistema de cursos online      | 15/08/2025     | Ana      |   

**Passos para execução do teste:**

1. Acessar a lista de projetos do empreendedor.
2. Selecionar um projeto cadastrado.
3. Visualizar os detalhes do projeto.
4. Editar as informações do projeto (ex.: alterar status ou descrição).
5. Clicar em “Salvar”.

**Resultado esperado:**

a) O sistema aplica as alterações corretamente.
b) O sistema exibe mensagem:

> **"Alterações realizadas com sucesso!"**

c) As alterações ficam registradas no sistema e disponíveis para futuras consultas.

**Cenário Alternativo:** Cancelamento da Edição

**Requisitos associados:**

* RF007: O sistema deve permitir que o empreendedor cancele a edição do projeto.
* RF008: O sistema não deve salvar alterações se o processo for cancelado.

**Preparação:**

O empreendedor acessa a lista de projetos e seleciona um projeto para edição.

**Passos para execução do teste:**

1. Editar algumas informações do projeto.
2. Clicar em “Cancelar” antes de salvar.

**Resultado esperado:**

a) O sistema interrompe o processo de edição.
b) Nenhuma alteração é salva no banco de dados.
c) O usuário permanece visualizando os detalhes originais do projeto.

**Cenário de Exceção 1:** Projeto Não Encontrado

**Requisitos associados:**

* RF009: O sistema deve exibir erro caso o projeto selecionado não exista ou tenha sido excluído.

**Preparação:**

O projeto selecionado foi excluído por outro processo ou não está mais disponível.

**Passos para execução do teste:**

1. Tentar acessar os detalhes de um projeto inexistente.

**Resultado esperado:**

a) O sistema exibe mensagem de erro:

> **"Projeto não encontrado."**

b) Nenhuma ação adicional é permitida até que o usuário selecione outro projeto.

**Cenário de Exceção 2:** Tentativa de Editar Projeto de Outro Usuário

**Requisitos associados:**

* RF012: O sistema deve impedir que projetos de outros usuários sejam editados ou excluídos.

**Preparação:**

O empreendedor deve estar logado no sistema
Deve existir pelo menos um projeto cadastrado que pertença a outro usuário (não ao empreendedor).
O empreendedor acessa a lista de projetos e tenta selecionar o projeto de outro usuário.

**Passos para execução do teste:**

1. Selecionar um projeto que pertença a outro usuário
2. Tentar editar ou excluir o projeto.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Você não tem permissão para editar ou excluir este projeto."**

b) Nenhuma alteração é realizada no projeto
c) O usuário permanece visualizando os detalhes do projeto, sem possibilidade de alteração.
