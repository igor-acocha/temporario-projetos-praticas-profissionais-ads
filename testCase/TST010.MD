 ## TST009 Gerenciar projetos

**Caso de uso que se baseia:** CDU010 Visualizar Lista de Interesses

**Cenário:** VVisualizar lista de estudantes interessados em projetos

**Requisitos associados:**

* RF001: O sistema deve permitir que o empreendedor visualize a lista de estudantes que demonstraram interesse em seus projetos.
* RF002: O sistema deve permitir que o empreendedor selecione um estudante para visualizar o perfil completo e detalhes da proposta.
* RF003: O sistema deve exibir mensagens de erro caso o estudante tenha retirado o interesse ou o projeto não esteja disponível.
* RF004: O sistema deve seguir regras de privacidade para informações sensíveis dos estudantes.

**Preparação:**

a) O empreendedor deve estar autenticado na plataforma.
b) Deve existir pelo menos um projeto com estudantes que tenham demonstrado interesse.
c) O sistema deve possuir registros de propostas ou interesses válidos.

**Dados de teste:**

| Estudante         | Projeto Interessado   | Status Interesse |            Proposta Resumida           |
| ------------------| ----------------------| -----------------| ---------------------------------------|
| Maria Silva       | App de Finanças       | Ativo            | Desejo colaborar no design UI          | 
| João Pereira      | Plataforma Educacional| Ativo            | Disponível para desenvolvimento backend|   
| Lucas Almeida     |Marketplace Online     | Retirou Interesse|            N/A                         |

**Passos para execução do teste:**

1. Acessar a área de Interesses do empreendedor.
2. Visualizar a lista de estudantes interessados em cada projeto.
3. Selecionar um estudante específico da lista.
4. Visualizar o perfil completo do estudante e detalhes da proposta.


**Resultado esperado:**

a) O sistema exibe corretamente todos os estudantes interessados.
b) O sistema apresenta o perfil completo do estudante selecionado e os detalhes da proposta.
c) O acesso às informações sensíveis segue as regras de privacidade da plataforma.

**Cenário Alternativo:** Nenhum Interesse Registrado

**Requisitos associados:**

* RF007: O sistema deve exibir mensagem adequada caso não haja estudantes interessados em um projeto.

**Preparação:**

O projeto selecionado não possui registros de estudantes interessados.

**Passos para execução do teste:**

1. Acessar a área de Interesses do projeto.

**Resultado esperado:**

a) O sistema exibe a mensagem:

"Nenhum estudante demonstrou interesse neste projeto até o momento."

b) O sistema não apresenta perfis de estudantes.

**Cenário de Exceção 1:** Estudante Removeu Interesse

**Requisitos associados:**

* RF009: O sistema deve exibir erro caso o estudante retire o interesse.

**Preparação:**

O estudante selecionado retirou o interesse pelo projeto.

**Passos para execução do teste:**

1. Tentar acessar o perfil do estudante que retirou o interesse.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Este estudante não está mais interessado neste projeto."**

b) Nenhuma informação adicional é carregada.

**Cenário de Exceção 2:** TProjeto Excluído

**Requisitos associados:**

* O sistema deve impedir a visualização de perfis de estudantes de projetos excluídos.

**Preparação:**

O projeto selecionado foi excluído.

**Passos para execução do teste:**

1. Tentar acessar a lista de estudantes vinculados ao projeto excluído.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Projeto não disponível. Perfis associados não podem ser visualizados."**

b) Nenhuma informação adicional é carregada

**Cenário de Exceção 2:** Erro de Carregamento

**Requisitos associados:**

* O sistema deve exibir mensagem apropriada caso haja falha no carregamento dos perfis.

**Preparação:**

Simular falha na recuperação dos dados do estudante.

**Passos para execução do teste:**

1. Tentar abrir o perfil de um estudante.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Não foi possível carregar o perfil. Tente novamente mais tarde"**

b) Nenhuma informação é exibida até a correção do erro.
