## TST007 Avaliar Usuário após Projeto

**Caso de uso que se baseia:** CDU007 Avaliar Usuário  

**Cenário:** Avaliação de Usuário com Dados Válidos

**Requisitos associados:**
- RF037: O sistema deve permitir que estudantes e empreendedores avaliem uns aos outros após a conclusão de um projeto.  
- RF038: O sistema deve validar os campos obrigatórios (nota e comentário).  
- RF039: O sistema deve impedir que um usuário avalie o outro mais de uma vez por projeto.  
- RF040: O sistema deve registrar a avaliação no perfil do usuário avaliado.  
- RF041: O sistema deve exibir mensagem de sucesso após o envio da avaliação.  
- RF042: O sistema deve impedir que o usuário saia da tela sem concluir a avaliação.  
- RF043: O sistema deve exibir aviso informando que a avaliação é obrigatória.  
- RF044: O sistema deve impedir envio se algum campo obrigatório estiver em branco.  

---

**Preparação**

a) O projeto deve estar concluído entre estudante e empreendedor.  
b) Ambos os usuários devem estar logados na plataforma.

**Passos para execução do teste**

1. O usuário acessa o projeto concluído na plataforma.  
2. Clica na opção “Avaliar Colaborador”.  
3. Preenche os campos obrigatórios: nota e comentário.  
4. Clica em “Salvar”.  

---

**Resultado esperado**

a) O sistema valida os dados e exibe a mensagem:

> **“Avaliação enviada com sucesso!”**

b) A avaliação é registrada no perfil do usuário avaliado.  
c) O sistema impede que o mesmo usuário realize nova avaliação para o mesmo projeto.  

---

**Cenário Alternativo: Saída sem Avaliar**

**Requisitos associados:**
- RF042: O sistema deve impedir que o usuário saia da tela sem concluir a avaliação.  
- RF043: O sistema deve exibir aviso informando que a avaliação é obrigatória.  

**Passos:**
1. Acessar a tela de avaliação.  
2. Tentar sair sem preencher os campos obrigatórios.  

**Resultado esperado:**  
a) O sistema exibe mensagem de alerta:  

> **“A avaliação é obrigatória. Complete o formulário antes de sair.”**

b) O usuário não consegue sair até preencher e enviar a avaliação.  

---

**Cenário de Exceção: Campos Obrigatórios em Branco**

**Requisitos associados:**
- RF038: O sistema deve validar os campos obrigatórios.  
- RF044: O sistema deve impedir envio se algum campo obrigatório estiver em branco.  

**Passos:**
1. Preencher parcialmente o formulário, deixando nota ou comentário em branco.  
2. Clicar em “Salvar”.  

**Resultado esperado:**  
a) O sistema exibe mensagem de erro:

> **“Preencha todos os campos obrigatórios antes de enviar a avaliação.”**

b) O envio não é realizado até que todos os campos estejam preenchidos.
