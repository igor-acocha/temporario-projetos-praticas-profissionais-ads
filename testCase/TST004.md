## TST004 Gerenciamento de Perfil de Usuário

**Caso de uso que se baseia:** CDU004 Gerenciar perfil

**Cenário:** Edição e Atualização de Perfil com Dados Válidos

**Requisitos associados:**

* RF018: O sistema deve permitir a edição das informações pessoais e profissionais do perfil.
* RF019: O sistema deve validar os campos editados antes de salvar.
* RF020: O sistema deve atualizar os dados e confirmar a operação com mensagem de sucesso.
* RF021: O sistema deve manter o perfil atualizado e disponível para futuras visualizações e edições.

**Preparação:**

a) O usuário deve estar logado na plataforma TechStart e possuir um perfil previamente criado.

b) O perfil inicial contém as seguintes informações:

| Campo                  | Valor Atual                                 |
| ---------------------- | ------------------------------------------- |
| Nome completo          | Enzo Henry da Rosa                          |
| E-mail                 | enzo.rosa@email.com.br                      |
| Tipo de usuário        | Estudante                                   |
| Telefone               | (11) 91234-5678                             |
| Área de interesse      | Desenvolvimento de Software                 |
| Descrição profissional | Estudante de tecnologia focado em inovação. |

**Passos para execução do teste:**

1. Acessar a seção “Meu Perfil” na plataforma TechStart.
2. Editar um ou mais campos do perfil (ex.: alterar telefone e área de interesse).
3. Clicar no botão “Salvar” para confirmar as alterações.

**Resultado esperado:**

a) O sistema valida as informações inseridas e exibe a mensagem:

> **"Perfil atualizado com sucesso"**

b) O sistema grava as novas informações e exibe o perfil atualizado na tela.

**Cenário Alternativo:** Cancelamento da Edição do Perfil

**Requisitos associados:**

* RF022: O sistema deve permitir ao usuário cancelar o processo de edição antes do salvamento.
* RF023: O sistema deve garantir que nenhuma alteração seja salva após o cancelamento.

**Preparação:**
O usuário deve estar logado e na tela de edição do perfil.

**Passos para execução do teste:**

1. Acessar a tela de edição de perfil.
2. Alterar um ou mais campos do formulário.
3. Clicar no botão “Cancelar”.

**Resultado esperado:**

a) O sistema interrompe o processo de edição.

b) Nenhuma informação é alterada no perfil existente.

c) O usuário é redirecionado de volta para a visualização do perfil original.

**Cenário de Exceção:** Informações Inválidas ou Campos Obrigatórios em Branco

**Requisitos associados:**

* RF024: O sistema deve validar todos os campos obrigatórios durante a edição.
* RF025: O sistema deve impedir a atualização caso haja informações inválidas ou faltantes.
* RF026: O sistema deve exibir mensagem de erro adequada ao problema identificado.

**Preparação:**
O usuário está logado e acessa a tela de edição do perfil.

**Passos para execução do teste:**

1. Deixar campos obrigatórios em branco ou preencher com dados inválidos (ex.: e-mail sem “@”).
2. Clicar no botão “Salvar”.

**Resultado esperado:**

a) O sistema exibe a mensagem de erro:

> **"Não foi possível salvar. Verifique os campos obrigatórios e tente novamente."**

b) O sistema não salva as alterações e mantém o usuário na tela de edição para correção.
