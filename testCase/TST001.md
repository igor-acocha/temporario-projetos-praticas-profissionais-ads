## TST001 Cadastro de Novo Usuário

**Caso de uso que se baseia:** CDU001 Cadastro de Usuário

**Cenário:** Cadastro de Usuário com Dados Válidos

**Requisitos associados:**

* RF001: O sistema deve permitir o cadastro de novos usuários na plataforma.
* RF002: O sistema deve validar os campos obrigatórios (nome, e-mail e senha).
* RF003: O sistema deve verificar se o e-mail informado já está cadastrado.
* RF004: O sistema deve registrar o novo usuário no banco de dados.
* RF005: O sistema deve enviar um e-mail de confirmação para o endereço informado.
* RF006: O sistema deve exibir mensagem de sucesso informando que o cadastro foi realizado.

**Preparação:**

a) O usuário não deve possuir cadastro prévio na plataforma.
b) O usuário deve possuir um endereço de e-mail válido e acesso à Internet.

**Passos para execução do teste:**

1. Acessar a página inicial da plataforma TechStart.
2. Clicar em “Cadastre-se”.
3. Preencher todos os campos obrigatórios: nome, e-mail e senha.
4. Clicar no botão “Cadastrar”.

**Resultado esperado:**

a) O sistema valida os dados e exibe a mensagem:

> **"Cadastro realizado com sucesso! Verifique seu e-mail para confirmação."**

b) O sistema envia um e-mail de confirmação para o endereço informado.
c) O novo usuário é registrado no banco de dados e aguarda a confirmação do e-mail.

**Cenário Alternativo:** Cancelamento do Cadastro

**Requisitos associados:**

* RF007: O sistema deve permitir ao usuário cancelar o processo de cadastro antes da conclusão.
* RF008: O sistema deve garantir que nenhum dado seja salvo em caso de cancelamento.

**Preparação:**
O usuário acessa a tela de cadastro.

**Passos para execução do teste:**

1. Preencher parcialmente os campos de cadastro.
2. Clicar em “Cancelar” ou fechar a página antes de concluir.

**Resultado esperado:**

a) O sistema interrompe o processo de cadastro.
b) Nenhuma informação é salva no banco de dados.
c) O usuário é redirecionado à tela inicial.

**Cenário de Exceção 1:** Dados Inválidos

**Requisitos associados:**

* RF009: O sistema deve validar o formato dos dados inseridos (e-mail, senha, nome).
* RF010: O sistema deve impedir o cadastro caso algum dado esteja incorreto.
* RF011: O sistema deve exibir mensagem de erro indicando os campos inválidos.

**Preparação:**
O usuário acessa a tela de cadastro e preenche dados incorretos.

**Passos para execução do teste:**

1. Informar um e-mail inválido (ex.: usuarioemail.com).
2. Inserir uma senha fraca (ex.: 12345).
3. Clicar em “Cadastrar”.

**Resultado esperado:**

a) O sistema exibe mensagem de erro:

> **"Verifique os campos destacados e tente novamente."**

b) O sistema não permite a continuação do cadastro.

**Cenário de Exceção 2:** E-mail Já Cadastrado

**Requisitos associados:**

* RF012: O sistema deve verificar se o e-mail informado já existe na base de dados.
* RF013: O sistema deve exibir mensagem informando que o e-mail já está cadastrado.

**Preparação:**
O e-mail informado já pertence a um usuário existente.

**Passos para execução do teste:**

1. Acessar a tela de cadastro.
2. Informar um e-mail que já existe na plataforma.
3. Preencher os demais campos e clicar em “Cadastrar”.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Este e-mail já está cadastrado. Deseja recuperar a senha?"**

b) O sistema não permite criar uma nova conta com o mesmo e-mail.

**Cenário de Exceção 3:** Falha no Envio de E-mail de Confirmação

**Requisitos associados:**

* RF014: O sistema deve tentar enviar o e-mail de confirmação após o cadastro.
* RF015: O sistema deve identificar falhas no envio do e-mail.
* RF016: O sistema deve exibir mensagem apropriada em caso de falha.

**Preparação:**
Simular falha de conexão ou serviço de e-mail indisponível.

**Passos para execução do teste:**

1. Realizar o cadastro normalmente com dados válidos.
2. Durante o processo, o sistema não consegue enviar o e-mail de confirmação.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Ocorreu um erro ao enviar o e-mail de confirmação. Tente novamente mais tarde."**

b) O cadastro é registrado no banco de dados, mas o status do e-mail permanece “pendente de confirmação”.
