## TST003 Criação de Perfil de Usuário

**Caso de uso que se baseia:** CDU003 Criar perfil

**Cenário:** Fluxo de Criação de Perfil com Dados Válidos

**Requisitos associados:**

* RF009: O sistema deve permitir que o usuário autenticado crie um perfil.
* RF010: O sistema deve validar todos os campos obrigatórios antes de salvar.
* RF011: O sistema deve armazenar as informações do perfil e exibir mensagem de sucesso após o salvamento.
* RF012: O sistema deve permitir apenas um perfil ativo por conta.

**Preparação:**

a) O usuário deve estar autenticado na plataforma TechStart (login realizado com sucesso).

b) Os campos disponíveis para criação de perfil são:

| Campo                  | Obrigatório | Exemplo de Preenchimento                    |
| ---------------------- | ----------- | ------------------------------------------- |
| Nome completo          | Sim         | Enzo Henry da Rosa                          |
| E-mail                 | Sim         | enzo.rosa@email.com.br                      |
| Tipo de usuário        | Sim         | Estudante                                   |
| Telefone               | Não         | (11) 91234-5678                             |
| Área de interesse      | Sim         | Desenvolvimento de Software                 |
| Descrição profissional | Não         | Estudante de tecnologia focado em inovação. |

**Passos para execução do teste:**

1. Acessar a área "Meu Perfil" dentro da plataforma TechStart.
2. Preencher todos os campos obrigatórios com informações válidas.
3. Clicar no botão “Salvar”.

**Resultado esperado:**

a) O sistema valida as informações e exibe a mensagem:

> **"Perfil criado com sucesso"**

b) O sistema armazena as informações e exibe o perfil recém-criado para visualização e edição futura.

**Cenário Alternativo:** Cancelamento da Criação de Perfil

**Requisitos associados:**

* RF013: O sistema deve permitir ao usuário cancelar a criação do perfil antes de salvar.
* RF014: O sistema não deve armazenar dados quando a criação é cancelada.

**Preparação:**
O usuário deve estar autenticado e na tela de criação de perfil.

**Passos para execução do teste:**

1. Acessar a tela de criação de perfil.
2. Preencher parcialmente ou totalmente os campos do formulário.
3. Clicar no botão “Cancelar”.

**Resultado esperado:**

a) O sistema interrompe o processo de criação.
b) Nenhuma informação é salva no banco de dados.
c) O usuário é redirecionado para a tela anterior (ex.: Painel principal).

**Cenário de Exceção:** Informações Inválidas ou Campos Obrigatórios em Branco

**Requisitos associados:**

* RF015: O sistema deve validar formato e obrigatoriedade dos campos.
* RF016: O sistema deve exibir mensagem de erro quando houver informações inválidas ou faltantes.
* RF017: O sistema deve impedir o salvamento até que os dados sejam corrigidos.

**Preparação:**
O usuário está logado e acessa a tela de criação de perfil.

**Passos para execução do teste:**

1. Acessar a tela de criação de perfil.
2. Deixar campos obrigatórios em branco ou preencher com dados inválidos (ex.: e-mail sem “@”).
3. Clicar no botão “Salvar”.

**Resultado esperado:**

a) O sistema exibe a mensagem de erro:

> **"Verifique os campos obrigatórios e tente novamente."**

b) O sistema não salva as informações e mantém o usuário na mesma tela para correção.
