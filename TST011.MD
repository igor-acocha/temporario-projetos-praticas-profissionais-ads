## TST009 Gerenciar projetos

**Caso de uso que se baseia:** CDU011 Gerenciar Plataforma

**Cenário:** Acessar painel administrativo e realizar ações de gerenciamento

**Requisitos associados:**

* RF001: O sistema deve permitir que o administrador acesse o painel administrativo com credenciais válidas.
* RF002: O sistema deve permitir que o administrador gerencie usuários, projetos e conteúdos.
* RF003: O sistema deve salvar alterações realizadas e atualizar o estado da plataforma.
* RF004: O sistema deve exibir mensagens de confirmação ou erro para cada ação realizada.
* RF005: Apenas administradores podem acessar funcionalidades administrativas.

**Preparação:**

a) O administrador deve possuir credenciais válidas com privilégios de acesso administrativo.
b) O sistema deve estar online e disponível para acesso.

**Dados de teste:**

|Administrador|	Usuário/Projeto/Conteúdo|  | Ação Realizada	Resultado Esperado         |
|-------------| ------------------------|  |-------------------------------------------|
|Felipe Admin |	Usuário João Silva	    |  | Alterar permissões	Permissões atualizadas |
|Ana Admin	  | Projeto App Finanças	|  | Excluir projeto	Projeto excluído       |  
|Carlos Admin |	Conteúdo Tutorial	    |  | Editar conteúdo	Conteúdo atualizado    |

**Passos para execução do teste:**

1. Administrador acessa a tela de login administrativo.
2. Insere credenciais válidas de administrador.
3. Sistema valida as credenciais.
4. Sistema redireciona para o painel de gerenciamento.
5. Sistema exibe opções de administração disponíveis (usuários, projetos, conteúdos).
6. Administrador seleciona a função desejada.
7. Sistema abre a tela correspondente à função escolhida.
8. Administrador realiza a ação de gerenciamento (editar, excluir ou atualizar).
9. Administrador confirma a alteração.
10. Sistema processa a alteração.
11. Sistema salva a alteração no banco de dados.
12. Sistema exibe mensagem de confirmação de sucesso.

**Resultado esperado:**

a) O sistema processa e salva corretamente todas as alterações realizadas.
b) O sistema exibe mensagem:

> **"Alteração realizada com sucesso!"**

c) As alterações ficam registradas no sistema e disponíveis para futuras consultas.

**Cenário Alternativo:** CAção Cancelada

**Requisitos associados:**

* RF006: O sistema deve permitir cancelar a ação de gerenciamento antes de salvar.
* RF007: Nenhuma alteração deve ser aplicada caso o processo seja cancelado.

**Preparação:**

Administrador acessa a função desejada e inicia uma ação de gerenciamento (edição ou exclusão).

**Passos para execução do teste:**

1. Realizar alterações em usuário, projeto ou conteúdo.
2. Clicar em “Cancelar” antes de confirmar.

**Resultado esperado:**

a) O sistema interrompe o processo de alteração.
b) Nenhuma alteração é aplicada no banco de dados.
c) O administrador permanece na tela com os dados originais.

**Cenário de Exceção 1:** Credenciais Inválidas

**Requisitos associados:**

* RF008: O sistema deve exibir erro caso o login ou senha do administrador estejam incorretos.

**Preparação:**

Administrador insere credenciais incorretas.

**Passos para execução do teste:**

1. Tentar acessar o painel administrativo com login ou senha incorretos.

**Resultado esperado:**

a) O sistema exibe mensagem de erro:

> **"Usuário ou senha inválidos."**

b) Nenhuma ação adicional é permitida até que o usuário selecione outro projeto.

**Cenário de Exceção 2:** Falha ao Salvar Alteração

**Requisitos associados:**

* RF009: O sistema deve exibir erro caso não consiga salvar alterações realizadas.

**Preparação:**

Simular falha de salvamento no banco de dados.

**Passos para execução do teste:**

1. Realizar uma ação de gerenciamento (edição, exclusão ou atualização).
2. Confirmar a alteração.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Não foi possível concluir a operação, tente novamente."**

b) Nenhuma alteração é realizada no projeto

**Cenário de Exceção 3:** Erro de Conexão

**Requisitos associados:**

* RF010: O sistema deve exibir mensagem apropriada caso a plataforma esteja indisponível.

**Preparação:**

Simular indisponibilidade do sistema.

**Passos para execução do teste:**

1. Administrador tenta acessar a plataforma.

**Resultado esperado:**

a) O sistema exibe a mensagem:

> **"Serviço temporariamente fora do ar, por favor tente mais tarde."**

b) Nenhuma alteração é realizada no projeto
